# 公文写作智能体 — 项目笔记

---

## 项目背景

### 问题痛点
大多数学生/基层公文写作者面临以下问题：
- **目的不明确**：写出来的是"流水账"，没有核心叙事
- **主体性缺失**：内容重点在"对方做了什么"，而非"我们收获了什么"
- **风格不统一**：不知道对标哪个央媒，导致写作调性混乱
- **缺少审稿机制**：写完无人审查，常见错误反复出现

### 项目定位
一个基于 Agentic Design Patterns 的**多智能体协作公文写作系统**，兼具**教学引导**与**生产辅助**双重功能。

---

## 写作指南核心心法

### 核心三问
公文写作不是记录"发生了什么"，而是回答：
1. 这件事证明了我们是谁？
2. 我们正走向何方？
3. 这件事对组织有什么价值？

### 五大写作原则
| 原则 | 说明 |
|------|------|
| **主体性原则** | 镜头始终对准"我们"，而非"对方" |
| **赋能性原则** | 每段行程回扣培养理念/战略部署 |
| **借势性原则** | 以外部权威（大师/平台/精神）为组织背书 |
| **成长性原则** | 用真实感言替代"大家纷纷表示" |
| **战略性原则** | 全文服务于组织长期发展，结尾传递信号 |

## 央媒风格调研

### 四大媒体风格对比

| 媒体 | 核心特点 | 适合场景 | 结构特征 |
|------|----------|----------|----------|
| **人民日报** | 庄重大气，政策站位高，宏大叙事 | 正式汇报、向上呈文 | 三段式：总-分-总 |
| **新华社** | 严谨凝练，信息密度大，数据与事实驱动 | 新闻通稿、对外发布 | 倒金字塔结构 |
| **央视新闻** | 场景化叙事，兼具深度与可读性 | 侧记、特写、人物通讯 | 故事线索贯穿 |
| **光明日报** | 理论性强，注重思想引领 | 调研报告、思想汇报 | 论点-论据-结论 |

## Agentic 设计模式映射

本项目遵循 [Agentic Design Patterns](https://adp.xindoo.xyz/)：

| 设计模式 | 项目对应实现 |
|----------|--------------|
| **Planning Pattern** | Questionnaire 模块：8 个结构化问题，提前规划写作策略 |
| **Reflection Pattern** | ReviewerAgent：独立审查 WriterAgent 输出，提供反馈与修改建议 |
| **Tool Use Pattern** | KnowledgeBase：内置范文库、错误模式库、术语库、过渡句库 |
| **Multi-Agent Collaboration** | Orchestrator 协调 Writer / Reviewer / StyleAdapter / KnowledgeBase |
| **Human-in-the-Loop (HITL)** | 两个关键节点让用户介入：<br>1) 确认写作简报后再开始<br>2) 审查终稿前可要求修改 |

## 灵魂拷问设计（8 题）

### 设计初衷
> 现在很多大学生写公文都没有目的性，结果像流水账。一开始应该有培训使用者和逼迫他们回答问题的机制。

### 问题设计思路

| 问题编号 | 核心问题 | 目的 |
|----------|----------|------|
| Q1 | 核心目的是什么？ | 避免流水账，明确写作目的 |
| Q2 | 第一读者是谁？（具体到人） | 为具体的人写作，语言才会精准 |
| Q3 | 还有哪些人可能读到？ | 满足多层读者需求 |
| Q4 | 这件事的深层含义是什么？ | 从新闻到公文的提炼 |
| Q5 | 与组织长期战略的关联？ | 体现顶层设计，非孤立事件 |
| Q6 | 有没有借势机会？ | 成为更大叙事的一部分 |
| Q7 | 手上有哪些不可替代的素材？ | 真实感言>空泛表态 |
| Q8 | 你的文章凭什么跟别人不一样？ | 逼迫找到独特视角 |

### 教学提示设计
每道题后附带"为什么这么问"的讲解，帮助用户理解背后的写作逻辑。

## AI Agent 能力评估标准

### 评估维度

| 维度 | 说明 | 本项目实现 |
|------|------|------------|
| **主动性** | Agent 是否能主动提问、引导用户思考 | ✅ Questionnaire 模块 |
| **规划性** | 是否有明确的执行计划和步骤 | ✅ Orchestrator 状态机 |
| **反思能力** | 能否对自身输出进行审查和改进 | ✅ ReviewerAgent 五轮审查 |
| **知识调用** | 是否能有效利用外部知识库 | ✅ KnowledgeBase |
| **风格适配** | 能否根据不同场景调整输出风格 | ✅ StyleAdapter |
| **人机协作** | 是否在关键节点让人类介入 | ✅ 双 HITL 节点 |
| **错误诊断** | 能否识别并指出具体问题 | ✅ 7 种错误模式匹配 |
| **可扩展性** | 架构是否支持后续功能扩展 | ✅ 模块化设计 |

## 系统架构

### 核心模块

| 模块 | 文件位置 | 职责 |
|------|----------|------|
| Questionnaire | `src/questionnaire/questionnaire.py` | 灵魂拷问模块，写作前引导 |
| StyleAdapter | `src/core/style_adapter.py` | 四大央媒风格适配 |
| DocTypeIdentifier | `src/core/document_type.py` | 五种文种自动识别 |
| WriterAgent | `src/core/writer_agent.py` | 内容生成 Agent |
| ReviewerAgent | `src/core/reviewer_agent.py` | 五轮审查 Agent |
| Orchestrator | `src/core/orchestrator.py` | 协调者 Agent，状态机管理 |
| KnowledgeBase | `src/knowledge/knowledge_base.py` | 知识库（范文/错误/术语/过渡句） |
| CLI | `cli.py` | 交互式入口 + QuickAPI |

### 完整流程

1. **规划阶段**：回答 8 道灵魂拷问，生成 WritingBrief
2. **HITL 确认 1**：用户查看、修改、确认简报
3. **风格与文种**：自动识别文种 + 选择媒体风格
4. **写作阶段**：WriterAgent 基于 Prompt 生成初稿
5. **五轮审查**：ReviewerAgent 按五大原则审查
6. **错误诊断**：匹配 7 种常见错误模式
7. **HITL 确认 2**：用户查看反馈，可修改或确认
8. **输出终稿**：终稿 + 写作简报 + 审查报告

## 技术决策记录

### 关键决策

| 决策 | 说明 | 原因 |
|------|------|------|
| Python 3.10+ | 使用现代 Python 语法和类型注解 | 类型安全 + 可读性 |
